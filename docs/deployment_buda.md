# Deploying to buda

This document describes the process for deploying the `xservice` application to the `buda` single-host environment.

## Prerequisites

1.  `buda` is a server running a recent version of Linux.
2.  [Docker](https://docs.docker.com/engine/install/) and [Docker Compose](https://docs.docker.com/compose/install/) are installed on `buda`.
3.  You have SSH access to `buda`.
4.  The `xservice` repository has been cloned to `buda`.

## Initial Setup

1.  **Create `.env` file**:
    On `buda`, navigate to the `xservice` directory and create a `.env` file. This file will contain the environment variables needed for production.

    **Security Note**: Do NOT use the `.env` file generated by the `scripts/bootstrap_local.sh` script. That script is for local development only. For production, you must create a new `.env` file with strong, unique credentials.

    Populate the `.env` file with the production values for the database and any other required secrets. For example:

    ```env
    DATABASE_URL=postgresql://xservice_user:a_very_strong_and_secret_password@postgres:5432/xservice_prod
    POSTGRES_USER=xservice_user
    POSTGRES_PASSWORD=a_very_strong_and_secret_password
    POSTGRES_DB=xservice_prod
    DEFAULT_COOKIE_FILE_PATH=/app/.secrets/default_cookies.txt
    DEFAULT_COOKIE_EXPECTED_COUNT=4
    ```

2.  **Create the default cookie secret file**:
    If you want the app to always keep four default X sessions available,
    create `/home/anchen/xservice/.secrets/default_cookies.txt` on `buda` and
    store one raw cookie string per line. The compose stack mounts `./.secrets`
    into the container read-only, so those secrets stay outside git while still
    being available at startup.

## Deployment Steps

1.  **Set the Host Port (Optional)**:
    The application's host port is configurable via the `XSERVICE_PORT` environment variable. If this variable is not set, it defaults to `8000`. To run the service on a different port (e.g., `8080`), you can export the variable before running Docker Compose:

    ```bash
    export XSERVICE_PORT=8080
    ```

2.  **Pull latest changes**:
    Ensure you have the latest version of the `main` branch.

    ```bash
    git pull origin main
    ```

3.  **Build and start the services**:
    Use Docker Compose to build the `app` image and start both the `app` and `postgres` services in detached mode. The command will pick up the `XSERVICE_PORT` from the environment.

    ```bash
    docker compose up --build -d
    ```

## Verifying the Deployment

You can check the status of the running containers:

```bash
docker compose ps
```

You can view the logs for the application:

```bash
docker compose logs -f app
```

The application should now be accessible at `http://<buda_ip>:${XSERVICE_PORT}` (e.g., `http://<buda_ip>:8000` if the variable is not set, or `http://<buda_ip>:8080` if you exported it as such).
